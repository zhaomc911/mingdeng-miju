# Legal Notes

This document records the intended intellectual-property boundaries for Mingdeng Miju (明灯迷局). It is a project note for development and publishing hygiene, not legal advice.

## Product Identity

- Product name: Mingdeng Miju / 明灯迷局.
- The project should not be branded as Nikoli, Akari by Nikoli, or any official Nikoli product.
- References to Akari or Light Up, if used, should be descriptive only, for example: "a logic puzzle inspired by classic Light Up / Akari rules".

## Game Rules

The game uses the general Light Up / Akari-style rule set:

- Bulbs illuminate white cells horizontally and vertically.
- Black walls block illumination.
- Numbered black walls constrain adjacent bulbs.
- A puzzle is solved when all white cells are illuminated, no bulbs see each other, and all numbered walls are satisfied.

The project treats these as general gameplay rules, methods, or mechanics. The project should avoid copying protected expression from third-party games, books, websites, apps, screenshots, or puzzle collections.

## Do Not Copy

Do not copy or closely recreate:

- Third-party puzzle layouts or level packs.
- Screenshots from existing puzzle websites or apps.
- Third-party UI, icons, illustrations, animations, fonts, sound effects, or music without a suitable license.
- Brand names, logos, marketing copy, tutorial text, or help pages from other products.

## Safe Development Practice

For future development:

- Generate campaign and infinite-mode puzzles with the project's own generator.
- Keep source notes for generated fixed levels.
- Use original UI, original naming, and either self-made or clearly licensed assets.
- If commercial launch, app store release, ads, payments, or multiplayer accounts are added, perform a dedicated legal and platform compliance review.

## Current Status

As of this note, the project is a client-side web puzzle game with locally generated/stored progress and no backend user account system.
